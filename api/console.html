üéÆ console ‚Äã

An implementation of the Web standard console API suitable for logging and debugging jstz smart functions.

DANGER

‚ö†Ô∏è jstz's implementation is not spec compliant ‚ö†Ô∏è

Quick start ‚Äã

Accessible from the global scope, console.log can be used to write a general logging message to the logs.

typescript
console.log("Hello from JS üëã"); // Hello from JS üëã

For messages with a more specific purpose the following methods are provided: console.info, console.warn, console.error, console.debug. For example:

typescript
function parseURL(arg: unknown) {
  console.debug(`Running parseURL(${arg})`);
  if (typeof arg !== "string") {
    console.warn(`Expected a string, but received ${typeof arg}`);
    arg = arg.toString();
  }
  try {
    let result = new URL(arg);
    console.info(`Result: ${result}`);
    return result;
  } catch (error) {
    console.error(`Failed to parseURL: ${error}`);
  }
}

Logging objects ‚Äã

When debugging jstz smart functions it is often useful to log the value of an object. A common mistake is to try log an object directly, for example:

typescript
console.log({ name: "Dave", age: 42 }); // [object Object]

This is because the jstz implementation of console doesn't support inspecting objects. A simple workaround is to use JSON.stringify:

typescript
const dave = { name: "Dave", age: 42 };
console.log(JSON.stringify(dave)); // { "name": "Dave", "age": 42 }

Assertions ‚Äã

console.assert will log an error message if its first argument is false. If the first argument is true there will be no effect.

typescript
function divide(a: number, b: number): number {
  console.assert(b != 0, "Trying to divide by 0");
  return a / b;
}

Groups ‚Äã

console.group facilitates the creation of nested log groups by introducing varying levels of indentation and assigning a group name. These groups can be neatly closed using console.groupEnd.

This feature is particularly handy for organizing logs to suit your needs. For instance, consider the following code, which conducts a brute force search for Pythagorean triples, showcasing the use of nested groups to reflect the level of organization within the logs.

typescript
function pythagoreanTriples(limit: number = 100) {
  for (let x = 2; x <= limit; ++x) {
    console.group(`x = ${x}`);
    for (let y = 1; y < x; ++y) {
      console.group(`y = ${y}`);
      for (let z = x; z < x + y; ++z) {
        console.log(`trying z = ${z}`);
        if (x * x + y * y === z * z) {
          console.log(`Success! ${x}^2 + ${y}^2 = ${z}^2`);
          break;
        }
      }
      console.groupEnd();
    }
    console.groupEnd();
  }
}

Running pythagoreanTriples(2) will produce the following output:

group: x = 2
  group: y = 1
    trying z = 2

Instance Methods ‚Äã

console.log(...message : unknown[]) : void ‚Äã

Outputs a general logging message. String representations of each of the arguments will be concatenated, separated by spaces and written to the logs.

console.info(...message : unknown[]) : void ‚Äã

Outputs a informative logging message. String representations of each of the arguments will be concatenated, separated by spaces and written to the logs.

console.warn(...message : unknown[]) : void ‚Äã

Outputs a warning logging message. String representations of each of the arguments will be concatenated, separated by spaces and written to the logs.

console.error(...message : unknown[]) : void ‚Äã

Outputs a warning message. String representations of each of the arguments will be concatenated, separated by spaces and written to the logs.

console.assert(assertion: unknown, ...message : unknown[]) : void ‚Äã

Outputs an error message if the first argument is falsy. String representations of each of the arguments will be concatenated, separated by spaces and written to the logs. Has no effect if the first argument is truthy.

console.debug(...message : unknown[]) : void ‚Äã

Outputs a debug logging message. String representations of each of the arguments will be concatenated, separated by spaces and written to the logs.

console.group(...label : unknown[]) : void ‚Äã

Begins a group and pushes the label to the group stack. The group label will consist of the string representations of each of the arguments, concatenated and separated by spaces. Subsequent messages will be indented at the level of the group.

console.groupCollapsed(...message : unknown[]) : void ‚Äã

This is provided for compatibility with existing frameworks. The behaviour is identical to console.group(...).

console.groupEnd() : void ‚Äã

Closes the current group and pops the group stack. Has no effect if the group stack is empty.

console.clear() : void ‚Äã

Provided for compatibility with existing frameworks. Closes all groups in the current group stack.
